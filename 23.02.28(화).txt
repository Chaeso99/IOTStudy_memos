# 스프링 개인적 정리
- 중간언어란?(XML, JSON)
어떤 언어를 완전히 학습하기 전까지 단계의 언어(불완전한 언어)
컴퓨터에서는 어셈블리어가 중간 언어라고 불림(고급 언어와 기계어 사이)

- MiME TYPE란?
멀티퍼포즈 인터넷 메일 익스텐션:파일 변환. 이메일과 함께 동봉할 파일을
텍스트 문자로 전환해서 이메일을 통해 전달하기 위해 개발되었음
현재는 웹을 통해 여러형태의 파일 전달에 쓰임

- REST API란?(개념, 메소드 등)
두 컴퓨터 시스템이 인터넷으로 정보를 안전하게 교환하기 위해 사용하는 인터페이스
REST는 Representational State Trasfer의 약자로 인터넷등의 복잡한 네트워크에서 통신을
관리하기 위한 지침이다.
메소드 종류:GET(자원 받아오기)/POST(여러개 가능, 새로운 자원 추가)
/PUT(단일 자원에만 가능, 존재하는 자원 변경)/DELETE(자원 삭제)/PATCH(한 자원 데이터
부분적 변경)

- HTTP Status란?(200, 400, 401, 403, 500)
HTTP 상태 코드이며 특정 HTTP 요청이 성공적으로 완료되었는지 알려줌
200:요청이 성공적으로 되었음
400:잘못된 문법으로 인해 서버가 요청을 이해할 수 없음
401:비인증(클라이언트가 스스로를 인증해야 요청에 대한 응답을 받을 수 있음)
403:클라이언트가 콘텐츠에 접근할 권리가 없음. 401과 다른점은 서버가 클라이언트가
누군지는 알고 있음
500:서버가 처리 방법을 모르는 상황(서버 에러)

- WS(Web Server)란?
HTTP 또는 HTTPS를 통해 웹 브라우저에서 요청하는 HTML 문서나 오브젝트(이미지 파일
등)을 전송해주는 서비스 프로그램

- WAS(Web Application Server)란?
웹 애플리케이션과 서버 환경을 만들어 동작시키는 기능을 제공하는 소프트웨어 프레임워크.
https://gmlwjd9405.github.io/2018/10/27/webserver-vs-was.html
ㄴWS와 WAS의 차이

- 톰캣이란?(수컷 고양이 라는뜻)
서블릿/JSP 및 HTTP 처리 엔진만 들어있는 웹 애플리케이션 서버
J2EE API를 추가하여 Enterprise 버전으로 배포중인 WAS인 Apache TomEE도 있다.

- 라이브러리란? -"책들의 집합"
소프트웨어를 개발할 때 컴퓨터 프로그램이 사용하는 비휘발성 자원의 모임
구성 데이터, 문서, 도움말 자료, 메세지 틀, 미리 작성된 코드, 클래스, 값 등이 포함됨
ㄴ라이브러리 사용 이유:모든 기능을 직접 코딩할수 없어서, 시간을 절약하기 위해서

- API란?
애플리케이션 프로그래밍 인터페이스의 약자로 다른 소프트웨어 시스템과 통신하기
위해 따라야 하는 규칙

- Framework란? Frame(틀) + work(일하다) = 일정한 틀, 뼈대를 가지고 일한다
IT에서는 특정 프로그램을 쉽게 만들기 위한 틀을 제공해주는 프로그램을 뜻한다.
예로 Spring, Django, Ruby on Rails 등이 있다.

- 라이브러리 / 프레임워크 비교
라이브러리는 도구의 모음-종류 엄청 다양함(쓰든 안쓰든 어떤걸 쓰든 상관없음)
프레임워크는 프로그램을 만들때 꼭 사용해야하는 요소와 메뉴얼에서 지켜야하는 규약
(자유도의 차이-프레임워크는 사용하는 이상 꼭 지켜야하는 규약이 존재
라이브러리는 쓰든 안쓰든 뭘쓰든~ 내맘임)

- SpringFramework란? 
https://khj93.tistory.com/entry/Spring-Spring-Framework%

자바 플랫폼을 위한 오픈 소스 애플리케이션 프레임워크. 간단히 스프링이라고 부름
동적인 웹 사이트를 개발하기 위한 여러 가지 서비스를 제공하고 있음.

- SpringFramework의 특징(IoC, DI, AOP 등등)
Inversion of Control(제어의 역전):객체는 자기가 사용할 객체를 선택하거나 생성하지 않음.
자신이 어디서 만들어지고 어떻게 사용될지 모름. 자신의 모든 권한을 특별한 다른 객체에
위임해서 제어권을 받은 특별객체에 의해 결정되고 만들어짐

Dependencty Injection(의존성 주입):객체가 서로 의존하는 관계가 되도록 의존성을 주입하는 것.
IOC에서의 DI는 각 클래스 사이에 필요로 하는 의존관계를 빈 설정 정보를 바탕으로 컨테이너가
자동으로 연결해 주는 것

Aspect Oriented Programming(관점 지향 프로그래밍):대부분의 SW 개발 프로세스에선 OOP를
사용하는데 OOP는 객체지향이라 비슷한 데이터를 한곳에 모아 분리하고 결합도가 낮아 중복된
코드들이 많아지고 가독성, 확장성, 유지보수성이 떨어짐. AOP에서는 핵심기능과 공통기능을
분리시켜 핵심 로직에 영향이 없게 공통기능을 끼워 넣는 개발형태를 사용함.
중복 코드 제거, 효율적인 유지보수가 가능해 재활용성이 극대화 되는 것이 장점.

- Spring Boot란?
스프링으로 애플리케이션을 만들 때 필요한 설정을 간편하게 처리해주는 별도의 프레임워크

- Spring Legacy와 Spring boot의 차이
기존 스프링을 Spring Legacy라고 함
Legacy에서는 프로젝트 설정을 직접 다 해야하지만 Boot는 많은 사람들이 사용하던 설정을
프리셋으로 구성해서 제공함. 받아서 나한테 필요한부분 조금 고치고 쓰면 된다
